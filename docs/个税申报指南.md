# 小规模纳税人企业个人所得税申报指南（首次申报）

> 适用场景：笑达公司首次为退休返聘人员申报个税，月薪 9,000 元
> 申报期限：次月15日前（如4月工资 → 5月15日前申报）

---

## 一、前期准备

### 1.1 需要的材料

| 材料 | 说明 |
|------|------|
| 员工身份证正反面复印件 | 用于人员信息采集 |
| 员工手机号 | 需能接收验证码 |
| 员工银行卡号 + 开户行 | 工资发放账户 |
| 退休证复印件 | 留档备查 |
| 退休返聘协议 | 已签订的劳务/返聘协议 |

### 1.2 需要的软件/账号

| 项目 | 说明 |
|------|------|
| **自然人电子税务局（扣缴端）** | 企业端申报软件，电脑安装 |
| 企业税务登记信息 | 统一社会信用代码、税务登记密码 |
| CA 证书或申报密码 | 登录扣缴端用 |

---

## 二、安装"自然人电子税务局（扣缴端）"

### 2.1 下载安装

1. 打开国家税务总局官网：https://etax.chinatax.gov.cn
2. 点击【下载中心】→ 找到【自然人电子税务局（扣缴端）】
3. 下载安装包，双击安装
4. 安装完成后桌面会出现图标

### 2.2 首次登录设置

1. 打开软件 → 选择【安装到本地/登录】
2. 输入公司的 **统一社会信用代码**（纳税人识别号）
3. 选择登录方式：
   - **申报密码登录**：使用在税务局设置的申报密码
   - **实名登录**：法定代表人或财务负责人扫脸认证
4. 首次登录会自动获取公司基本信息，确认无误后进入主界面

---

## 三、人员信息采集（仅首次需要）

> 这一步是告诉税务局"你公司有哪些人拿工资"

### 3.1 操作步骤

1. 打开扣缴端 → 左侧菜单点【人员信息采集】
2. 点左上角【添加】按钮

### 3.2 填写人员信息

按以下内容逐项填写：

| 字段 | 填写内容 | 说明 |
|------|----------|------|
| 证件类型 | 居民身份证 | |
| 证件号码 | 填员工身份证号 | 18位 |
| 姓名 | 填员工真实姓名 | 与身份证一致 |
| 国籍（地区） | 中国 | |
| 是否残疾/烈属/孤老 | 否 | 如实填写 |
| 手机号码 | 填员工手机号 | |
| **人员状态** | **正常** | |
| **是否雇员** | **是** | ⚠️ 退休返聘按雇员处理，享受工薪所得扣除 |
| **任职受雇日期** | **2026-04-01** | 入职日期 |
| 入职年度就业情形 | 新入职且累计无收入 | 如果他今年之前没有在其他单位领过工资 |

### 3.3 报送验证

1. 填写完成后点【保存】
2. 选中该人员 → 点上方【报送】按钮
3. 等待几秒 → 状态变为 **"报送成功"** 或 **"验证通过"**
4. 如果显示"验证不通过"，检查身份证号和姓名是否与身份证完全一致

> ⚠️ 报送成功后，该人员信息就正式登记到税务局了。

---

## 四、综合所得申报（每月操作）

### 4.1 进入申报界面

1. 左侧菜单点【综合所得申报】
2. 点【正常工资薪金所得】右边的【填写】按钮

### 4.2 填写工资数据

在工资表中找到你添加的员工，填写以下字段：

| 字段 | 填写内容 | 说明 |
|------|----------|------|
| **本期收入** | **9000** | 税前工资 |
| 本期免税收入 | 0 | |
| 基本养老保险 | 0 | 退休人员不缴社保 |
| 基本医疗保险 | 0 | |
| 失业保险 | 0 | |
| 住房公积金 | 0 | |
| 累计子女教育 | 0 | 除非他有符合条件的专项附加扣除 |
| 累计住房贷款利息 | 0 | |
| 累计住房租金 | 0 | |
| 累计赡养老人 | 0 | |
| 企业年金 | 0 | |
| 商业健康保险 | 0 | |
| 其他扣除 | 0 | |

### 4.3 系统自动计算

填完收入后，系统会自动计算：

```
第1个月（4月工资）：
  累计收入           = 9,000.00
  累计减除费用       = 5,000.00（每月固定）
  累计专项扣除       = 0.00
  累计专项附加扣除   = 0.00
  累计应纳税所得额   = 9,000 - 5,000 = 4,000.00
  累计应纳税额       = 4,000 × 3% = 120.00
  累计已预缴税额     = 0.00（首月）
  本期应补（退）税额 = 120.00
```

> 确认 **本期应补（退）税额 = 120.00 元** 即为正确。

### 4.4 提交申报

1. 核对数据无误后，返回【综合所得申报】主页面
2. 点右下角【税款计算】→ 系统汇总计算
3. 检查计算结果：应纳税额 120 元
4. 点【申报表报送】→ 确认 → 点【发送申报】
5. 等待几秒 → 显示 **"申报成功"**

---

## 五、税款缴纳

### 5.1 缴税方式

申报成功后需要缴纳税款：

1. 在申报成功页面点【立即缴款】（或左侧菜单【税款缴纳】）
2. 选择缴款方式：
   - **三方协议缴款**：如果公司已签订银税三方协议，直接从对公账户扣款（推荐）
   - **银行端缴款**：生成缴款凭证，去银行柜台缴纳

### 5.2 确认缴款成功

- 缴款后状态变为 **"缴款成功"**
- 可以在【查询统计】→【申报情况查询】中查看历史记录

---

## 六、后续每月操作（5月~12月重复）

每月只需重复第四步和第五步：

### 6.1 每月申报流程

```
1. 打开扣缴端
2. 选择税款所属期（如：2026年5月）
3. 综合所得申报 → 正常工资薪金所得 → 填写
4. 本期收入填 9000
5. 税款计算 → 核对
6. 申报表报送
7. 税款缴纳
```

### 6.2 每月预扣税额变化（全年一览）

由于累计预扣法，每月的应税额一致但累计数在增长：

| 税款所属期 | 本期收入 | 累计收入 | 累计减除 | 累计应税 | 累计税额 | **当月扣税** |
|:---:|---:|---:|---:|---:|---:|---:|
| 4月 | 9,000 | 9,000 | 5,000 | 4,000 | 120 | **120** |
| 5月 | 9,000 | 18,000 | 10,000 | 8,000 | 240 | **120** |
| 6月 | 9,000 | 27,000 | 15,000 | 12,000 | 360 | **120** |
| 7月 | 9,000 | 36,000 | 20,000 | 16,000 | 480 | **120** |
| 8月 | 9,000 | 45,000 | 25,000 | 20,000 | 600 | **120** |
| 9月 | 9,000 | 54,000 | 30,000 | 24,000 | 720 | **120** |
| 10月 | 9,000 | 63,000 | 35,000 | 28,000 | 840 | **120** |
| 11月 | 9,000 | 72,000 | 40,000 | 32,000 | 960 | **120** |
| 12月 | 9,000 | 81,000 | 45,000 | 36,000 | 1,080 | **120** |

> 全年均在 3% 税率区间，每月固定扣税 **120 元**，全年合计 **1,080 元**。

| 5月工资 | 5月底或6月初 | **6月15日** | 2026年5月 |

## 八、常见问题

### Q1：4月入职，前三个月（1-3月）要申报吗？
**不需要。** 该员工4月入职，从4月的税款所属期开始申报。1-3月公司如果没有其他员工，继续零申报即可。

### Q2：每月发工资时要扣税吗？
**是的。** 实际发放工资时就应扣除个税：
- 应发工资：9,000 元
- 代扣个税：120 元
- 实发工资：**8,880 元**（银行转账这个金额）

### Q3：扣下来的120元什么时候交给税务局？
次月15日前申报并缴纳。4月的120元，在5月15日前通过扣缴端申报并缴款。

### Q4：退休人员需要做年度汇算清缴吗？
月薪9000、全年应纳税所得额36000，每月预扣都是3%，**不需要年度汇算**。全年预扣税额 = 应纳税额 = 1080元，不存在多缴少缴。

### Q5：之前的零申报怎么办？
继续保持。1-3月仍按零申报（无人员工资发放），4月开始有人员有税额。扣缴端会自动区分。

### Q6：需要给员工开工资条吗？
建议每月提供工资明细（应发、扣税、实发），可以用纸质或电子版，员工签字确认存档。
